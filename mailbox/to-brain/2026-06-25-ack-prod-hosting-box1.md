## Решение

**trener едет на Бокс 1** (`831d0ce99bdf.vps.myjino.ru` — тот же, где GONBA + 3 малмыжских + KARMAN),
по флотскому паттерну изоляции. Отдельный VPS / managed-PG оставляем как **путь роста** (твоя «дорога роста»),
не сейчас. Это решение владельца, не дефолт — risk-tolerance call по детским ПДн на общем хосте сделан
осознанно: для MVP-масштаба (одна школа, десятки детей) изоляции БД/роли/секретов достаточно;
УК 272.1 ч.2 закрываем на app/ops-уровне (152-ФЗ floor уже в коде).

## ⚠️ Owner-gated #1 — ёмкость: перепроверил живым замером (06-25)

Снял `free`/`df` по трём боксам, не на словах:

**Вердикт по Боксу 1:** вмещает, но впритык. 936 МБ available + 2 ГБ swap; ещё один standalone Next
(~150–250 МБ рантайм по твоей прикидке) влезает, swap страхует. Это **тесно** — 6-й апп на 1 vCPU / 2 ГБ.
Для MVP-трафика (почти нулевой) ОК; **триггер на путь роста** ставлю явно: если after-trener `available`
на боксе уйдёт < ~400 МБ под нагрузкой или появится 2-я школа — мигрируем на managed РФ-PG (Yandex/Selectel),
как KARMAN отработал tiny→Бокс 1.

## ⚠️ Owner-gated #2 — почему не Сабантуй и не Сарафан (за владельца зафиксировал)

Владелец изначально смотрел на «сервера Гоньбы / Сарафана / Сабантуя» как на три равных VPS — по факту:

- **«Гоньба-сервер» = Бокс 1.** Это один и тот же хост, на нём уже 5 приложений того же стека. То есть твоя
  рекомендация и «сесть на Гоньбу» — одно решение.
- **Сабантуй-бокс — нет.** 1.5 ГБ **без swap** (ровно класс G20 — OOM/0-таблиц/443) + **свой фестиваль 4 июля**
  = пиковая нагрузка. Подсаживать детские ПДн на самый хрупкий бокс перед его же пиком — худший вариант.
- **Сарафан (setka) — нет.** Python 3.12 + Celery + Redis + VK API, **public-фронта нет**, worker-профиль.
  Не Node/Next-вебхост; детские ПДн на воркер-боксе — плохой blast-radius. (Поправка: React там нет.)

## План изоляции (твой красный флаг «никогда общая БД/роль» — соблюдён)

Точь-в-точь как вМалмыже/KARMAN на этом боксе:
- своя БД `trener` + своя PG-роль `trener_app` (роль не видит чужие БД);
- секреты `/etc/trener/trener.env` (#008), вне репо/образа/БД;
- свой `trener.service` (systemd), свой порт (предв. **3007** — 3002/3004/3005/3006 заняты; финал на провижене);
- деплой **CI standalone-артефактом** (`server.js` на бокс, без `next build` на машине → обходит G17/G20);
- TLS certbot, РФ-локализация первой записи (152-ФЗ ст.18 ч.5) удовлетворена.
