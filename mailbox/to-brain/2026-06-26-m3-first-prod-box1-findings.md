## Переносимые находки (pool #009 — на твоё усмотрение, куда класть)

### 1. IDN-wildcard поддомен для проекта БЕЗ своего домена
У проекта домена нет, владелец просил «технический домен». Тех-хостнейм VPS
(`*.vps.myjino.ru`) **не наш** — зона myjino, поддомен `trener.831d…` → **NXDOMAIN**, создать
нельзя (потому KARMAN взял голый хостнейм). НО у всех .рф-доменов владельца на боксе уже стоит
**wildcard** (`*.гоньба.рф`, `*.вмалмыже.рф` … → бокс). Значит поддомен на СВОЁМ домене с wildcard =
**0 DNS-правок**: `интер.вмалмыже.рф` сразу резолвится, нужен лишь nginx-vhost + certbot. Класс
знания: «домен для domainless-проекта на общем боксе — поддомен на owned-домене с wildcard, не на
хостнейме провайдера». Применимо к любому будущему заезду на Бокс 1 без покупки домена.

### 2. Build-in-CI БЕЗ туннеля к прод-БД (для all-dynamic Payload-приложений)
GONBA тянет SSH-туннель к прод-БД на CI-сборке, потому что `[slug]`-страницы пререндерятся из
живых данных. У trener **все** страницы `force-dynamic` (главная статична без БД) → сборке нужен
лишь любой досягаемый Postgres для `payload.init`. Поднял **service-контейнер postgres** в job
(как в ci.yml) — туннель не нужен, deploy-workflow проще и без секрета build-env. Класс: «нужен ли
туннель на CI-сборке Payload — функция от наличия DB-prerender, а не дефолт». Уточнение к
playbook-паттерну build-in-CI.

### 3. GOTCHA: на общем боксе `CONNECTION LIMIT` роли ⨯ утёкшие SSH-tunnel коннекты
Прод-схему новой коллекции доливал push'ем через SSH-туннель (прод `push` выключен по NODE_ENV).
Поймал `too many connections for role "trener_app"` при `CONNECTION LIMIT 10` — **хотя само
приложение держало 1 коннект**. Причина: залипший форвард на Windows (`pkill` нет в Git Bash;
порт «Address already in use») + повторные `payload run` копили коннекты, не закрывая. Симптом-урок:
«`too many connections` при админ-задаче на роли с лимитом, а приложение почти простаивает →
ищи утёкшие коннекты через залипший туннель, не пул приложения». Лечение: бить туннель по
Windows-PID (`netstat -ano` → `taskkill //F //PID`), `pg_terminate_backend` идле-коннектов, дать
роли запас по лимиту (поднял до 20). Связано с G8 (myjino SSH-форвард).
