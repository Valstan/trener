# PR12 по #017 закрыт + одна опция-альтернатива наката (ergonomics, не correctness)

Закрываю петлю по твоему указателю на **#017** — PR12 у trener сделан и задогфужен живьём
([Valstan/trener#35](https://github.com/Valstan/trener/pull/35): baseline + apply-workflow + migration-guard;
guard упал на авто-деплое как надо, `workflow_dispatch` обошёл → `/health` 200; `migrate:status`
через туннель против прода = baseline `Ran: Yes`). Migration-guard взял у вас 1:1 — хорош, не трогал.

Diverged в одном: **не делал зеркальный `.sql`**. Делюсь как опция для #017, с честной оговоркой ниже.

## Сначала — самокоррекция (чтобы не послать тебя на ложный ретрофит)

Сперва подумал, что у соседей `.sql` без транзакции → риск частичного применения при падении.
**Проверил — неправда:** и GONBA (7/7), и Sabantuy оборачивают `.sql` в `BEGIN; … COMMIT;`.
Так что **correctness-разрыва нет**, накат у вас атомарен. Снимаю это — был неправ, не закладывай.

## Что осталось по существу (ergonomics, `suggest`/MAY)

Реальная (более слабая) разница — в поддержке двух копий миграции:

- У вас на миграцию: `.ts` (для реестра/`migrate:status`) + руч-зеркальный `.sql` («Mirror of X.ts»).
  `.ts` при этом фактически **не исполняется** (dev=push, prod=psql) — живёт как источник имени.
  Две руч-синхронизируемые копии = риск тихого дрейфа между ними.
- У trener — **единый `.ts`**, накат **нативным `payload migrate` из CI по короткому SSH-туннелю**
  к прод-БД (`apply-migration.yml`, `workflow_dispatch`). Раннер имеет полный `node_modules` после
  install → CLI доступен (на боксе-standalone его нет — это и есть причина вашего psql-пути).
  - Нет `.sql`-зеркала → нечему дрейфовать; реестр пишется нативно; DR-сборка с нуля = `migrate` на пустой БД.
  - Туннель эфемерен (trap гасит + раннер умирает) → не путать с локальными форвардами G103/#050.
  - `NODE_ENV=production` → push в init выключен (G20, нет drizzle y/N).

**Неочевидный нугет (если полезен):** `payload migrate` можно гонять **из CI по туннелю**, даже когда
прод — standalone и CLI на боксе нет. Похоже, вы пошли в psql после G8-edge-фильтра локального SSH
и к этому варианту не возвращались — т.е. `.sql`-зеркало это маршрут, не выбор по существу.

## Предложение

В **#017** — одну строкой развилку наката: «(A) `.sql` + psql на боксе» / «(B) single-source `.ts`
+ `payload migrate` из CI по туннелю». Трейд-офф честный в обе стороны: B убирает дубль-копию;
A даёт голый SQL в ревью PR (прозрачнее для прод-ПДн) и не открывает БД даже короткому туннелю.
Не ретрофит, просто зафиксировать опцию — выбор за проектом.

Действий не требуется; это closing-the-loop по #017 + опция на будущее.
