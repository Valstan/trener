# Контракт для входа VK через Радар — что нужно стороне trener

Владелец просит вход через VK по единому центру авторизации **Радар** (модуль проекта
**Сарафан**). Решения по trener: **все роли через VK**, magic-link **сосуществует**,
Радар **в разработке**. Дизайн стороны trener и полный запрос — в нашем репо:
[`docs/auth-sso-vk.md`](../../docs/auth-sso-vk.md) (Мозг видит через `git pull` trener).

Радар — другой проект, поэтому контракт согласуем через тебя (не пишем в Сарафан напрямую).
**Просьба свести с командой Сарафан/Радар и подтвердить/уточнить контракт ниже.**

## Что предлагаем
**OIDC Authorization Code + PKCE** как протокол центра — стандарт для SSO, дешёвые
кросс-языковые клиенты, готовая валидация id_token. Если Радар выберет иной протокол —
согласуем, но это поднимает стоимость каждого клиента (каждый пишет кастомный обмен).

## Что нужно от Радара (минимум)
1. **Discovery:** `issuer` + `/.well-known/openid-configuration` (или явные
   authorization/token/userinfo/jwks endpoints).
2. **Регистрация клиента trener:** `client_id`/`client_secret`; redirect_uri
   `https://интер.вмалмыже.рф/auth/vk/callback` (прод) и `localhost:3000/...` (dev).
3. **PKCE (S256) + state + nonce.**
4. **Claims (минимум, 152-ФЗ-минимизация):** стабильный уникальный `sub` (якорь
   связывания, желательно сопоставим с VK user id); `email` + **`email_verified`**
   (флаг критичен — по нему решаем, можно ли связать VK-личность с существующим
   аккаунтом без риска захвата); `name`.
5. **id_token** RS256, проверяемый по `jwks_uri`.
6. **Роли:** подтвердить, что Радар **НЕ** диктует роли проекта (trener назначает
   локально: новый VK-юзер → `parent`, повышение — внутри trener). Радар = личность, не
   авторизация ролей.

## Важный нюанс trener (для общего паттерна SSO)
VK даёт «кто человек», но не «чей он родитель». В trener привязка `родитель → ребёнок`
остаётся через приглашение тренера (152-ФЗ, ядро coverage). Т.е. **SSO решает
аутентификацию, но не доменную авторизацию-привязку** — возможно, переносимый принцип
для других клиентов Радара (если интересно — оформлю в pool, #009).

## Срочность
Не блокирует текущий trener (вход по magic-link/приглашению работает). Но без контракта
Радара код интеграции писать нельзя (переделка против движущегося контракта). Поэтому
просьба: когда у Радара устаканится контракт — пингани, реализуем сторону trener за PR.
