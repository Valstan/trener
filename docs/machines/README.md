# docs/machines/ — профили окружения на каждый компьютер

[pool #050](../../../brain_matrica/cross-project-ideas/ideas/050-per-machine-env-profiles.md). Разработка trener ведётся поочерёдно с двух машин (работа / дом). Один файл на hostname: `<hostname>.md` — порты dev-стенда, пути к инструментам, рабочие команды, машинные грабли.

- **Читается** `/start`'ом по hostname текущей машины (узнал бокс → подхватил порт/пароль dev-БД, нюанс pnpm и т.п., не нащупывая заново).
- **Пишется** по мере изучения локального окружения. Выяснил факт → записал строкой.
- **Правит свой файл только сам этот комп** — межмашинных merge-конфликтов нет.
- **Коммитится** обычным PR-flow → переживает реклон, синхронизируется между машинами через git (GitHub = источник истины, [#010](../../../brain_matrica/cross-project-ideas/ideas/010-session-sync-safeguard.md)). Чужой профиль = справка, не трогаем.
- **⚠️ Без секретов** — порты/пути/команды да, пароли/ключи/токены НЕТ ([#008](../../../brain_matrica/cross-project-ideas/ideas/008-secrets-outside-repo.md)). Секреты живут в `web/.env` (gitignored); профиль лишь ссылается «пароль — в web/.env».
- **Окружение, не проект.** Сюда — специфику железа/настройки. Проектное состояние (нитки/техдолг) — в `SESSION_HANDOFF` / `PENDING_FOLLOWUPS`.

## Известные машины

| hostname | роль | WORKSPACE_ROOT | профиль |
|---|---|---|---|
