## Dev-БД (PostgreSQL) — критично для auth/flow-смоука

- Служба **`postgresql-x64-17`** (PostgreSQL 17), запускается вручную (Manual). Data-dir `C:\Program Files\PostgreSQL\17\data`.
- **Порт `5433`** (НЕ 5432 — на 5432 ничего не слушает). `pg_hba`: `scram-sha-256` (пароль обязателен).
- Суперюзер **`postgres`**; пароль = **тот же, что у GONBA/Sabantuy** (см. их `web/.env`, и наш `web/.env`) — НЕ строка «postgres». Пароль в профиль не пишем (#008).
- БД проекта: **`trener_dev`** (создана на 5433). `web/.env` (gitignored) → `DATABASE_URL=postgresql://postgres:<pwd>@127.0.0.1:5433/trener_dev`.
- **psql НЕ в PATH** → полный путь: `"C:\Program Files\PostgreSQL\17\bin\psql.exe" -h 127.0.0.1 -p 5433 -U postgres -d trener_dev` (пароль через `PGPASSWORD`).
- Схему материализует `corepack pnpm -C web dev` (payload `push:true` при init). Воссоздать БД с нуля: `CREATE DATABASE trener_dev;` на 5433 → `pnpm dev`.

## Машинные грабли

- **Preview-инструмент НЕ сохраняет httpOnly-cookie из fetch** → авторизованную страницу (`/parent`, `/coach`) скриншотом в превью не снять. Обход: `curl` с реальной сессией (логин через `/auth/complete-login`, cookie-jar) — проверять рендер HTML + состояние в БД. SW в dev не регистрируется (только prod) → web-push-кнопку локально не протестировать (это M3/HTTPS).
- **Force-push запрещён правами** (даже в feature-ветку в этой сессии отклонялся) → фиксы поверх запушенного PR делать **новым коммитом**, не `--amend`+force.
- **EOL-шум:** генераторы (`payload generate:types`, dev-сервер) переписывают `payload-types.ts` / `importMap.js` с CRLF без смысловых правок → перед коммитом `git checkout -- <файл>`, если `git diff --ignore-all-space` пуст.
