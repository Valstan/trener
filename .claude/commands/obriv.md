Был **обрыв интернета/сессии**. Не паникуй и **не переделывай вслепую то, что уже сделано** — сначала восстанови картину из «земли» (git/файлы/PR), потом продолжи прерванную нить.

## 1. Реконструируй состояние из ground truth

```bash
git branch --show-current
git status --short
git log --oneline -6
git stash list
```
- Рабочее дерево чистое? Есть незакоммиченные правки — они **завершённые или оборванные на полпути**?
- Последний коммит — это та работа, что я делал перед обрывом, или она не успела закоммититься?

## 2. Сверь с PR и фоновыми задачами

```bash
gh pr view --json number,state,title,headRefName -q '.number,.state,.title' 2>/dev/null || gh pr list --head "$(git branch --show-current)"
```
- PR существует/открыт/смержён? Не задвоить создание.
- Были ли фоновые задачи (dev-сервер, build)? Проверь их вывод; не плодить дубли на :3000.

## 3. Проверь целостность недавно записанных файлов (грабля харнесса)

На этой машине (Windows) обрыв может совпасть с **битой записью файла**: в файл попадает лишний `NUL`-байт (или UTF-16/BOM) → git показывает файл как **`Bin`** в `git diff --stat`, хотя сборка может «проглотить».

```bash
git add -A && git diff --cached --stat   # любой исходник как "Bin" → подозрение
```
Если код-файл показан бинарным — проверить и вычистить NUL, пересохранить UTF-8:
```bash
node -e "const fs=require('fs');const f=process.argv[1];const b=fs.readFileSync(f);const n=[...b].filter(x=>x===0).length;console.log(f,'NUL',n,'utf8',(()=>{try{new TextDecoder('utf-8',{fatal:true}).decode(b);return true}catch{return false}})());if(n)fs.writeFileSync(f,Buffer.from([...b].filter(x=>x!==0)))" '<путь>'
```

## 4. Реконсиляция последнего действия

Сопоставь **последнее, что я намеревался сделать** (из контекста диалога), с фактическим состоянием файла: правка реально легла? Если не уверен — **перечитай** нужный участок файла (Read), не доверяй памяти.

## 5. Если трогался код — перепроверь гейты

```bash
corepack pnpm -C web typecheck && corepack pnpm -C web lint
```
(и `corepack pnpm -C web build`, если правка существенная).

## 6. Доложи и продолжи

Кратко: `После обрыва: ветка <X>, последний коммит <Y>, дерево <чисто/N правок>, PR #<N> <состояние>. Потеряно: <ничего/…>. Продолжаю: <следующий шаг>.` — и **продолжай прерванную работу** с того места, где она встала.
